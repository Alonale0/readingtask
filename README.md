## 四、遇到的问题及解决方法（不少于2个）
 
问题1： git commit  命令执行后出现  >  箭头，无法正常提交
 
- 问题描述：输入  git commit -m "第二次提交，添加注释"  后按回车，终端持续出现  >  符号，无法完成提交操作。

- 原因分析：提交说明的双引号未闭合，Git 认为命令未结束，处于多行输入等待状态。

- 解决方法：

1. 按  Ctrl + C  退出当前输入状态，回到命令行提示符；

- 问题描述：本地  git commit  命令执行成功，终端显示提交记录，但 GitHub 远程仓库页面始终只有第1次提交记录，无后续更新。
